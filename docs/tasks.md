# KORA 介绍网站 - Tasks

> 拆分日期：2026-07-02
> 规则：每完成一个 task → 测试 → 确认 → commit → 下一个

## Phase 1：页面骨架

### Task 1.1 创建 index.html 基础结构

- **目标**：搭建 HTML 骨架，包含所有 6 个 Section 的语义结构和文字内容
- **涉及文件**：`index.html`
- **执行步骤**：
  1. 创建 HTML5 文档结构（lang="zh-CN"）
  2. 引入 `css/style.css` 和 `js/main.js`
  3. 依次编写 6 个 `<section>`，填入 PRD 中的完整文案
  4. 语义化标签：header、section、footer
- **验收标准**：直接打开 index.html 能看到完整文字内容，结构清晰
- **测试方式**：浏览器打开，检查 6 个部分内容完整

### Task 1.2 创建 CSS 基础变量与布局

- **目标**：建立 CSS 变量体系、基础排版和响应式骨架
- **涉及文件**：`css/style.css`
- **执行步骤**：
  1. CSS 自定义属性（颜色、字体、间距、圆角）
  2. 基础 reset 和排版（字体、行高、颜色）
  3. 各 section 的 max-width 居中布局
  4. 移动端 < 768px 的基础适配
- **验收标准**：页面在任何宽度下内容居中、可读
- **测试方式**：Chrome DevTools 切换手机/平板/桌面，检查排版

### Task 1.3 创建 JS 滚动框架

- **目标**：搭建 Intersection Observer 基础框架，为后续动效做准备
- **涉及文件**：`js/main.js`
- **执行步骤**：
  1. 初始化 Intersection Observer（threshold: 0.2）
  2. 为每个 section 添加 data-animate 属性
  3. 进入视口时添加 `.visible` class
  4. 确保动画只触发一次
- **验收标准**：滚动页面时，每个 section 进入视口时会在控制台打印日志
- **测试方式**：打开 DevTools Console，滚动验证

## Phase 3：动效核心 - Section 3 对话演示

### Task 3.1 对话气泡 HTML 结构

- **目标**：在 Section 3 中搭建完整的对话气泡 DOM 结构
- **涉及文件**：`index.html`
- **执行步骤**：
  1. 用户消息气泡（右侧，英文提问原文）
  2. "KORA 正在输入..." 三个跳动点的气泡
  3. 3 条餐厅推荐回复气泡（左侧）
  4. 1 条补充提醒气泡
- **验收标准**：HTML 结构完整，每条气泡内容包含评分/地址/链接/司机用语
- **测试方式**：浏览器查看静态结构（此时还没动效）

### Task 3.2 气泡基础样式

- **目标**：对话气泡的视觉样式
- **涉及文件**：`css/style.css`
- **执行步骤**：
  1. 用户气泡样式（右对齐、蓝色/灰色背景）
  2. KORA 气泡样式（左对齐、浅色背景）
  3. 气泡内信息层级（店名粗、评分彩色、地址灰色）
  4. 补充提醒气泡特殊样式（淡黄色边框）
  5. 模仿 Telegram 聊天气泡风格
- **验收标准**：像真实聊天界面，信息层级清晰
- **测试方式**：视觉对比 Telegram 截图风格

### Task 3.3 气泡弹出动效

- **目标**：实现气泡逐条出现的动效
- **涉及文件**：`js/main.js`, `css/style.css`
- **执行步骤**：
  1. CSS @keyframes：气泡从侧边滑入 + 淡入
  2. JS 控制气泡出现时序：
     - 用户气泡先出现（延迟 0.3s）
     - "..." typing indicator 出现并跳动（持续 1.5s）
     - 三条餐厅气泡依次出现（每条间隔 0.5s）
     - 提醒气泡最后出现
  3. 整个动效在 Section 3 进入视口时触发
- **验收标准**：动画流畅，时序自然，60fps
- **测试方式**：多次滚动到 Section 3 位置，观察动画

### Task 3.4 打字效果（可选增强）

- **目标**：KORA 回复气泡的文字以打字机效果逐字出现
- **涉及文件**：`js/main.js`
- **执行步骤**：
  1. 实现 typewriter 函数（逐字输出）
  2. 在气泡滑入后触发打字
  3. 可配置打字速度
- **验收标准**：文字以自然速度逐字出现
- **测试方式**：观察打字速度是否自然

## Phase 4：全局动效与打磨

### Task 4.1 滚动淡入动效

- **执行步骤**：

- **测试方式**：滚动到 Section 4，观察卡片出现顺序
