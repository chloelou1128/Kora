# KORA 介绍网站 - Design

> 基于 PRD v1，2026-07-02

## 1. 系统架构

纯静态单页网站，无后端、无构建工具、无框架依赖。浏览器直接打开 `index.html` 即可运行。

```
浏览器
  └── index.html
        ├── css/style.css     ← 全局样式 + 响应式 + 动效
        ├── js/main.js        ← 滚动触发 + 气泡动画 + 打字效果
        └── assets/           ← 图片、图标等静态资源
```

## 3. 动效方案

### 3.1 技术选型

全部用纯 CSS + Vanilla JS 实现，不引入动画库。
- **CSS Animations**：气泡弹出、淡入、滑入
- **Intersection Observer**：检测 section 进入视口，触发动画
- **JS 定时器**：模拟打字效果、气泡逐条出现

### 3.2 Section 3 对话动效（核心）

这是页面最重要的动效区域，模拟真实的 Telegram 聊天体验：

**阶段 1：用户消息气泡**
- 右侧弹出，灰色/蓝色气泡
- 显示英文提问内容
- 动画：从右侧滑入 + 淡入，约 0.3s

**阶段 2：KORA 思考中**
- 左侧出现 "..." 跳动的 typing indicator（三个点依次跳动）
- 持续约 1-1.5s

**阶段 3：KORA 回复气泡逐条出现**
- 每条餐厅推荐是一个独立气泡
- 气泡内容包含：店名、评分、人均、风格说明、地址、导航链接、司机沟通语
- 动画：从左侧滑入 + 淡入，每条间隔约 0.5s
- 气泡内部小元素（评分、标签）有微动效

**阶段 4：补充提醒气泡**
- 最后一条淡黄色/灰色提示气泡
- 内容：营业时间说明、支付方式提醒

### 3.3 其他 Section 动效

- **Section 1**：标题文字逐字淡入（或整体 fade-in-up）
- **Section 2**：左右对比内容从两侧分别滑入
- **Section 4**：三个卡片依次从下方浮入（stagger 0.2s）
- **Section 5**：地图/城市标记点动画（如有视觉素材）
- **Section 6**：三步流程依次出现

### 3.4 滚动触发策略

```
Intersection Observer
  threshold: 0.2          ← 元素 20% 可见时触发
  once: true              ← 只触发一次，避免重复播放
```

## 4. 响应式策略

### 断点

| 断点 | 目标设备 |
|------|----------|
| < 768px | 手机（主要场景） |
| 768px - 1024px | 平板 |
| > 1024px | 桌面 |

### 移动端优先规则

- 字体最小 15px，保证微信内置浏览器可读
- 对话气泡最大宽度 85vw（手机）/ 600px（桌面）
- Section 间距充裕，保证滚动阅读节奏
- 标签和按钮 touch target ≥ 44px
- 图片和动效在弱网下优雅降级

## 6. 部署方案

**目标**：GitHub Pages

**步骤**：
1. 创建 GitHub 仓库
2. Push 代码到 `main` 或 `master` 分支
3. 在仓库 Settings → Pages 中启用 GitHub Pages
4. 选择部署分支和根目录
5. 访问 `https://<username>.github.io/Kora/`

由于是纯静态 HTML，无需任何构建步骤。Push 即部署。

## 7. 关键风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 设计风格未对齐 | 开发完成后需大量返工 | 先与 Codex 确认设计方向，UI 部分先不做 |
| 动效性能 | 低端手机卡顿 | 使用 CSS transform/opacity（GPU 加速），避免 JS 动画 |
| 微信浏览器兼容 | 部分 CSS 特性不支持 | 关键样式做渐进增强，动效降级为静态展示 |
| 对话气泡内容过长 | 移动端显示拥挤 | 气泡最大宽度限制，长文本合理换行 |
